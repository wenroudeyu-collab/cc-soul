Python 怎么读取大 CSV 文件？内存不够用
cc
用 pandas.read_csv 的 chunksize 参数分块读取：

for chunk in pd.read_csv('big.csv', chunksize=10000):
  process(chunk) 顺便说一下：
• 如果只需要部分列，加 usecols=['col1','col2'] 能再省 60% 内存
• 文件超过 10GB 建议换 polars（Rust 写的，比 pandas 快 5-10x）
• 注意编码问题——中文 CSV 经常是 GBK 不是 UTF-8，不指定 encoding 会乱码
• 如果数据最终要入库，csv → SQLite → 查询 比全加载到内存更可靠

纠错验证 — 你说"你错了"，它不会盲目道歉
👤
你说错了，TCP 三次握手是为了确认双方的收发能力，不是为了防止重复连接
cc
我来验证一下这个纠正。

你说的"确认双方收发能力"是三次握手的效果，但不是它存在的根本原因。

RFC 793 原文明确说明三次握手的设计目标是："to prevent old duplicate connection initiations from causing confusion"（防止已失效的连接请求报文段突然又传送到服务端）。

两个说法不矛盾，但因果关系是反的：
✗ 你的版本 三次握手为了确认收发能力
✓ 准确版本 三次握手为了防止旧重复连接，顺带确认了收发能力
这次我不改我的结论。但你的理解也不算错，只是因果关系需要调一下。
💡 触发方式：自动。当你纠正 bot 时（"你错了/不对/搞错了"），bot 不会盲目道歉，而是先验证你的纠正是否正确，再决定接受还是礼貌反驳。

mkdir -p ~/.openclaw/plugins/cc-soul/data/models/minilm
cd ~/.openclaw/plugins/cc-soul/data/models/minilm
curl -L -o model.onnx "https://huggingface.co/Xenova/all-MiniLM-L6-v2/resolve/main/onnx/model_quantized.onnx"
curl -L -o tokenizer.json "https://huggingface.co/Xenova/all-MiniLM-L6-v2/resolve/main/tokenizer.json"
# 2. 安装 ONNX Runtime（向量计算引擎）
cd ~/.openclaw/plugins/cc-soul
npm install onnxruntime-node
